# SoloSales.OS Build & Deployment Guide

This guide provides comprehensive instructions for building, deploying, and monetizing the SoloSales.OS platform. It is designed to be a complete reference for replicating the current state of the application.

## 1. Architecture & Tech Stack

SoloSales.OS is a full-stack web application built with a modern, lightweight stack:

*   **Frontend:** React 18, TypeScript, Vite, Tailwind CSS, React Router DOM, React Hot Toast (for notifications).
*   **Backend:** Node.js, Express.js.
*   **Database:** SQLite (using `better-sqlite3` for synchronous, high-performance local data storage).
*   **Authentication:** Custom JWT (JSON Web Tokens) with `bcryptjs` for password hashing. Cookies are used for secure session management.
*   **AI Integration:** Google Gemini API (`@google/genai` SDK) for generating sales assets (ICP, email sequences, LinkedIn DMs, discovery scripts, objection handling).

## 2. Project Structure

The project is organized into a single repository containing both frontend and backend code:

*   `server.ts`: The main Express.js server entry point. Handles API routes, authentication, database interactions, and serves the frontend in production.
*   `db.ts`: SQLite database initialization and schema definitions.
*   `src/`: Frontend React source code.
    *   `api.ts`: Utility for making authenticated API calls to the backend.
    *   `App.tsx`: Main React component and routing configuration.
    *   `context/AuthContext.tsx`: React Context for managing user authentication state globally.
    *   `pages/`: React components for each page (Login, Signup, Dashboard, Project, Onboarding, Settings).
    *   `pages/components/`: Reusable UI components (KanbanBoard, Questionnaire, AssetsView).
*   `public/`: Static assets.
*   `dist/`: Production build output (generated by Vite).

## 3. Environment Variables

Create a `.env` file in the root directory with the following variables:

```env
# Required for AI Asset Generation
GEMINI_API_KEY=your_gemini_api_key_here

# Required for JWT Authentication (Change this in production!)
JWT_SECRET=your_super_secret_jwt_key_here

# Optional: Port for the server (defaults to 3000)
PORT=3000
```

## 4. Local Development Setup

1.  **Install Dependencies:**
    ```bash
    npm install
    ```

2.  **Start the Development Server:**
    ```bash
    npm run dev
    ```
    This uses `tsx` to run `server.ts`. The Express server will start on port 3000, and it uses Vite middleware to serve the React frontend with Hot Module Replacement (HMR).

3.  **Access the App:** Open your browser and navigate to `http://localhost:3000`.

## 5. Production Build & Deployment

The application is designed to be deployed as a single Node.js service.

### Build Process

1.  **Build the Frontend:**
    ```bash
    npm run build
    ```
    This command uses Vite to compile the React frontend into static files located in the `dist/` directory.

### Deployment (e.g., Render, Railway, DigitalOcean App Platform)

1.  **Environment Setup:** Ensure your hosting provider supports Node.js and persistent storage (for the SQLite database file, typically `database.sqlite` in the project root).
2.  **Environment Variables:** Set `GEMINI_API_KEY`, `JWT_SECRET`, and `NODE_ENV=production` in your hosting provider's dashboard.
3.  **Start Command:** Configure your hosting provider to use the following start command:
    ```bash
    npm start
    ```
    *(Note: Ensure your `package.json` has `"start": "node --loader ts-node/esm server.ts"` or compile `server.ts` to JavaScript first if your environment doesn't support running TypeScript directly in production).*

    **Important Note on SQLite in Production:**
    SQLite stores data in a local file (`database.sqlite`). If you deploy to an ephemeral file system (like Heroku or standard Vercel serverless functions), your database will be reset on every deployment or server restart. You **must** use a hosting provider that offers persistent disk storage (e.g., Render with a Disk attached, Railway with a Volume, or a standard VPS) to ensure data persistence.

## 6. Monetization Strategy (Stripe Integration)

To monetize SoloSales.OS, integrate Stripe for subscription management.

### Step 1: Stripe Setup
1.  Create a Stripe account.
2.  Create a Product (e.g., "SoloSales.OS Pro") and a recurring Price (e.g., $29/month).
3.  Get your Stripe Secret Key and Webhook Secret.

### Step 2: Backend Integration (`server.ts`)
1.  Install Stripe: `npm install stripe`
2.  Add routes to create Stripe Checkout Sessions:
    ```typescript
    import Stripe from 'stripe';
    const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);

    app.post('/api/checkout', authenticateToken, async (req, res) => {
      const session = await stripe.checkout.sessions.create({
        payment_method_types: ['card'],
        line_items: [{ price: 'price_YOUR_PRICE_ID', quantity: 1 }],
        mode: 'subscription',
        success_url: \`\${req.headers.origin}/dashboard?success=true\`,
        cancel_url: \`\${req.headers.origin}/dashboard?canceled=true\`,
        client_reference_id: req.user.id.toString(), // Link payment to user
      });
      res.json({ url: session.url });
    });
    ```
3.  Add a Webhook endpoint to listen for successful payments and update the user's subscription status in the SQLite database.

### Step 3: Frontend Integration
1.  Add a "Upgrade to Pro" button in the UI (e.g., in `SettingsPage.tsx` or a dedicated Billing page).
2.  When clicked, call the `/api/checkout` endpoint and redirect the user to the returned Stripe Checkout URL.
3.  Update the UI to restrict premium features (like generating assets) to users with an active subscription status.

## 7. Core Features Implemented

*   **Custom Authentication:** Secure email/password signup and login using JWT and bcrypt.
*   **Onboarding Flow:** Collects essential business information to tailor the AI generation.
*   **Project Management:** Users can create and manage multiple sales projects.
*   **AI Asset Generation:** Integrates with Google Gemini to automatically generate ICPs, email sequences, LinkedIn DMs, discovery scripts, and objection handling guides based on project details.
*   **Kanban Lead Board:** A drag-and-drop interface for managing sales leads through different stages (To Contact, Contacted, Meeting Booked, Proposal Sent, Closed Won, Closed Lost).
*   **Settings & Account Management:** Users can update their password or delete their account (which cascades to delete their projects, assets, and leads).

## 8. Troubleshooting & Common Errors

*   **"Failed to generate assets" (Gemini API):** Ensure your `GEMINI_API_KEY` is correct and has sufficient quota. Check the server console for detailed error logs from the `@google/genai` SDK.
*   **Authentication Errors (401/403):** Verify that the `JWT_SECRET` is set correctly. Ensure cookies are being sent with requests (the frontend `fetchApi` utility handles this automatically by including credentials).
*   **Database Errors:** If you encounter SQLite errors, ensure the application has write permissions to the directory where `database.sqlite` is created.
*   **Vite HMR Errors:** If you see WebSocket connection errors in the browser console during development, ensure `hmr: false` is set in `vite.config.ts` if running in a constrained environment, or verify your port configurations.
